PUT Update Order Items

PUT /orders/{id}/items

⚡ Order Modification with Inventory Delta Calculation 🆕

Update order items with smart inventory management:

  • Validates: Only PENDING orders can be modified
  • Calculates Delta: Compares old vs new items to determine exact inventory changes
  • Updates Inventory: Increments/decrements based on delta
  • Recalculates Total: Computed Pattern updates order total
  • ACID Transaction: All changes atomic or rollback
💾 ACID Transactions 🧮 Computed Pattern 🎯 Inventory Delta

💡 Delta Calculation Example

Old: Laptop(2), Mouse(3) → New: Laptop(3), Mouse(1), Keyboard(1)
Deltas: Laptop -1, Mouse +2, Keyboard -1

Response

POST Cancel Order

POST /orders/{id}/cancel

🚫 Order Cancellation with Inventory Restoration 🆕

Cancel an order with full ACID transaction guarantees:

  • Validates: Only PENDING/CONFIRMED orders can be cancelled
  • Restores Inventory: All items returned to stock atomically
  • Updates Status: Changes to CANCELLED with reason tracking
  • Audit Trail: Complete history of cancellation
  • ACID Transaction: Either all succeeds or all fails (no partial cancellations)
💾 ACID Transactions 🧾 Audit Trail 📦 Embedding Pattern

💡 Transaction Rollback Example

If inventory restoration fails (e.g., product deleted), the entire cancellation is rolled back. Order remains in original state!

📊 MongoDB Aggregation Framework - Revenue Analysis 🆕

Demonstrates MongoDB's powerful aggregation pipeline:

  • $match: Filter out cancelled orders (like SQL WHERE)
  • $group: Group by status and calculate totals (like SQL GROUP BY)
  • $sum: Aggregate revenue and count orders
  • $sort: Order by revenue descending
📊 Aggregation Pipeline 💰 Business Intelligence 🧮 Computed Pattern

Executive dashboards, revenue reporting, order pipeline analysis, business metrics
